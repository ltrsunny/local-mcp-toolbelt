1. RACE CONDITIONS / CONCURRENCY (‚Č§120 words)
In `runner.ts` (lines 86-89), `job.args` is passed by reference if `thinking_resolved === undefined`, and only shallow-copied via spread (`{ ...job.args }`) otherwise. If the wrapped tool mutates nested objects within `args` (e.g., JSON schemas or arrays), it permanently corrupts the in-memory cached job state. Concurrent jobs sharing the same underlying args, or a retried job reusing the metadata, will see corrupted input. This is not thread-safe. A deep clone is required across the invocation boundary.
