# v0.5.0 shipped — final state (2026-05-10)

This is the **canonical post-mortem** for v0.5.0. Three earlier scope memos
(`v0.4.0-llama-cpp-backend`, `v0.4.0-larger-models-eval`,
`v0.5.0-tier-d-eval`) tracked the iterative path; they were deleted
because their amended decision sections contradicted each other. Only this
file describes what actually shipped.

---

## What changed (vs v0.4.0)

**Project rename**: `ollama-mcp-bridge` → `local-mcp-toolbelt`.
Binary: `local-mcp`. Package: `local-mcp-toolbelt`.

**Backend collapse to single oMLX path**:
- Removed: `OllamaBackend`, `OllamaClient`, `OllamaDaemonError`,
  `LlamaCppBackend`, `node-llama-cpp` dep, `ollama` dep, `model?` /
  `keepAlive?` / `modelPath?` fields on `TierConfig`.
- Single backend in v0.5.0: `MlxHttpBackend` (`src/llm/mlx-http-backend.ts`).
- Inference engine: oMLX (`jundot/omlx`, brew service, port 8000).
- All three tiers route to oMLX via OpenAI-compatible
  `/v1/chat/completions`, distinguished only by `mlxModelName` + `numCtx`.

**Tier configuration** (`DEFAULT_CONFIG` in `src/config/tiers.ts`):

| Tier | Model | numCtx | Notes |
|------|-------|--------|-------|
| B | `Qwen3-4B-Instruct-2507-4bit` | 8 192 | Non-thinking variant. Default for short tools. |
| C | `Qwen3-8B-4bit` | 32 768 | Thinking model + bridge auto-injects `/no_think`. Long-form summarize. |
| D | `Qwen3-14B-4bit` | 16 384 | Same `/no_think` injection. Opt-in for `classify` + `transform`. |

**Schema enforcement**: `MlxHttpBackend` sends
`response_format: {type: "json_schema", strict: true}` and
`normalizeForStrictMode()` patches every object node with
`additionalProperties: false` + full `required` lists so oMLX's strict
mode actually engages (verified 2026-05-07: enum constraints binding,
schema fields produced; without normalization oMLX silently falls back
to unconstrained mode).

**Per-tool output caps** (`MAX_OUTPUT_TOKENS` in `src/mcp/server.ts`,
mirrored in `tests/eval/lib/invoke.mjs`):

| Tool | Cap | Note |
|------|-----|------|
| `summarize` | 600 | 1-2 paragraphs |
| `summarize-long` | 1 200 | structured summary, ≤6 bullets |
| `classify` | 200 | labels + brief reason |
| `transform` | 1 200 | instruction-driven |
| `extract` | 2 048 | schema-driven |
| `diff-semantic-index` | 1 024 | structured fields |

**Prompt thinking control**: `MlxHttpBackend.chat()` appends `\n/no_think`
to the user message. Qwen3 thinking models honour the token to disable
the `<think>...</think>` reasoning trace; non-thinking variants
(Qwen3-4B-Instruct-2507) treat it as inert. Without this token, thinking
variants exhaust the per-tool cap on reasoning before producing output.

**Hook enforcement** (commit `b2c8316`): `.claude/settings.json` +
`.claude/hooks/enforce-bridge.sh` ship with the repo. PreToolUse on
`Read | Bash` blocks direct reads of external files >1 KB; stderr injects
the bridge tool name + `source_uri` pattern back into Claude's context.
Allow-list: project tree, `~/.claude/`, `~/.omlx/`, files ≤1 024 bytes.
Reader vocabulary (cat/head/tail/awk/sed/grep/jq/...) only triggers at
command position; redirect targets are stripped before path scan.

**MCP server name**: `claude_desktop_config.json` key is
`local-mcp-toolbelt` (was `ollama-bridge`). Tools surface as
`mcp__local-mcp-toolbelt__*`.

**oMLX cache**: `~/.omlx/settings.json` configured with
`hot_cache_max_size: "4GB"` + `ssd_cache_dir: "~/.omlx/cache"`. Bridge
calls reuse prefix KV cache across requests at the oMLX layer.

## What did not ship in v0.5.0

- **Tier D promotion in `toolTierMap`**: classify + transform are
  Tier-D-routable per eval data (commit `8b0251b`), but `DEFAULT_CONFIG`
  keeps them on B; user opts in via config override.
- **Validated 4B-Instruct eval**: the Tier B switch from 8B to
  4B-Instruct was made on a single curl test (3.6 s for classify) plus
  the strict-mode + `/no_think` interaction. Project's own promotion
  criteria (5-trial 95 p latency, quality avg ≥4.0) were NOT met for
  the current B tier. **Tracked for v0.6.0 Phase 1 eval re-run.**
- **Output-side bridge enforcement**: input compression is hook-enforced;
  output generation (`transform` tool for boilerplate) remains
  voluntary. Tracked for v0.7.0 consideration after Phase 1 data.
- **Security hardening**: `file://` sandbox, weight signing, mTLS,
  `npm install` postinstall audit. Out of scope per
  `v0.5.0-threat-model-2026-05-10.md`; tracked for parallel v0.6.0
  Phase 3 scope memo.

## Test status

- 147 unit tests pass via vitest. CI in-process only (uses
  `RecorderBackend` test double via `_installTestBackend`). No real oMLX
  required.
- `tests/smoke-bridge.mjs` (the v0.4.0 Ollama smoke harness) was
  deleted with the Ollama backend. **No replacement integration test
  ships in v0.5.0.** Tracked for v0.6.0 Phase 1 (eval-as-smoke pattern).

## Process notes

This was a 3-phase brainstorm cycle (R1 / R2 / R2-fanout). The R2-fanout
final tally was 5:1 in favor of "Type-A docs first → eval → Type-B
docs", with one rider from `mistralai/mistral-large-3-675b` to prepend
a tiny threat-model memo (this scope memo is partly that rider). Voices
that engaged: gem (Gemini 3.1 Pro), Claude (orchestrator), a fresh
Claude subagent, oMLX 14B local, mistral-large-3, qwen3.5-397b. Voices
that failed or off-topic: copilot (kept answering security audit),
deepseek-v4-pro (NIM 504 capacity), kimi-k2.6 (tokenizer corruption).
