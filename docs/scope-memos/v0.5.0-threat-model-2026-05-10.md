# Threat model — v0.5.0 (2026-05-10)

`local-mcp-toolbelt` is a **single-user, single-machine personal toolbelt**.
The owner installs via `git clone + npm install`, runs the bridge process
under their own UID, and is the sole writer of `/tmp/`, `~/.omlx/models/`,
and the bridge stdout. There is no multi-tenant deployment.

## In scope (defended)

- **Prompt injection in source content**: `summarize` / `extract` /
  `classify` route untrusted text through `BridgeDefense` (Spotlighting +
  `@stackone/defender` Tier 1; optional MiniLM ONNX Tier 2).
- **Schema escape via crafted JSON Schema**: `sanitizeSchemaForStrictMode`
  strips constraints that crash decode (`pattern`, `format`, `$ref`).
- **Frontier-token leakage**: `enforce-bridge.sh` PreToolUse hook blocks
  direct reads of external files >1KB; route via `source_uri`.

## Out of scope (accepted risk, single-user)

- **Local code execution**: anything with shell access on rd's machine
  already trumps the toolbelt's defenses.
- **`npm install` postinstall script attacks**: rd vets the repo before
  cloning; standard supply-chain hygiene.
- **HF model weight tampering**: `npm run download-models` fetches from
  `mlx-community/*` over plain HTTPS without signature verification.
  Acceptable while single-user; tracked separately for v1.0 publish.
- **oMLX server compromise**: bridge calls `http://127.0.0.1:8000` with
  no mTLS or auth. Localhost-only and rd-controlled.

## Deferred to security scope memo (v0.6.0 Phase 3, parallel track)

- `file://` source_uri sandbox (allowlist defender-controlled paths)
- HF weight signature verification on `npm run download-models`
- `npm install` postinstall audit checklist
- Optional mTLS on bridge → oMLX channel for multi-user mode

These are tracked in a separate scope memo, not blocking v0.6.0 docs/eval.

## Implications for documentation

Type-A docs (architecture, install, hook, CHANGELOG of what shipped)
**may assert the single-user assumption**. Type-B docs (per-tier model
recommendations, latency claims) **must wait for eval re-run**. Future
multi-user docs are explicitly out of v0.6.0 scope.
